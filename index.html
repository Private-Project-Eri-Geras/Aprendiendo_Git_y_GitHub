¿Qué es Git?

Git es un software de control de versiones pensando en la eficiencia y la confiabilidad del mantenimiento de versiones de aplicaciones cuando estas tienen un gran número de archivos de código fuente.

Al usar Git, los cambios realizados en el código fuente se registran en un repositorio, lo que permite rastrear el historial completo de cambios realizados. Además, Git facilita la colaboración entre desarrolladores al permitirles trabajar en diferentes ramas del código fuente sin interferir entre sí.

Aunque existen algunas herramientas gráficas para Git, la línea de comandos sigue siendo la herramienta más poderosa y versátil para realizar tareas avanzadas de Git.

¿Qué es GitHub?

GitHub es una plataforma en línea basada en Git que amplía las capacidades del control de versiones y facilita la colaboración en proyectos de desarrollo de software.

En esencia, GitHub actúa como un repositorio en la nube donde los desarrolladores pueden almacenar y gestionar sus proyectos utilizando Git.

Al usar GitHub, los equipos pueden acceder y colaborar fácilmente en el código fuente de manera distribuida. La plataforma proporciona características adicionales, como seguimiento de problemas, solicitudes de extracción y herramientas visuales para facilitar la revisión de código. Esto hace que la gestión de proyectos sea más eficiente y transparente.

Comandos de Git

Clone

Crea una duplicación local de un repositorio remoto. Este comando es esencial cuando deseas obtener una copia de un proyecto existente para colaborar o trabajar en él de forma individual.

$ git clone url_del_repositorio

init

Inicializa un nuevo repositorio Git en tu directorio local. Este comando marca el inicio de un nuevo proyecto y establece el seguimiento de cambios en tu código.

$ git init

status

Proporciona información sobre el estado actual del repositorio, incluyendo archivos modificados, en el área de preparación y sin seguimiento.

$ git status

add

Agrega archivos al área de preparación de Git, lo que significa que los cambios en esos archivos se incluirán en el próximo commit.

$ git add ejemplo.txt

Para agregar todos los archivos modificados usamos:

$ git add -A

commit

Guarda los cambios en tu repositorio local. Cuando haces un commit, agregas una descripción de los cambios que has hecho. Se usa con la opción -m seguida de un mensaje que describa los cambios realizados.

$ git commit -m “Bug arreglado”

pull

Descarga los cambios realizados en el repositorio remoto a tu repositorio local y los combina con tus cambios locales. Es útil cuando otros colaboradores han realizado cambios en el repositorio y necesitas actualizar tu copia local.

$ git pull

push

Sube los cambios realizados en tu repositorio local al repositorio remoto. Es útil para compartir tus cambios con otros colaboradores y asegurarte de que las versiones estén sincronizadas.

$ git push

branch

Crea, lista o borra ramas en tu repositorio. Las ramas son útiles para trabajar en características separadas sin afectar la rama principal.

$ git branch nueva_rama

merge

Combina los cambios de una rama en otra. Útil para integrar el trabajo realizado en una rama con la rama principal.

$ git merge rama_a_combinar

rebase

Aplica los cambios de una rama sobre otra, reescribiendo la historia del proyecto. Útil para mantener un historial de commits más lineal.

$ git rebase otra_rama
